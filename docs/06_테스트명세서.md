## 1. RAG 유사도 분포 평가

### 1.1 목적
실제 상담 발화가 RAG 검색 임계값(0.6) 이상으로 잘 검색되는지 확인하여,
시스템의 검색 커버리지를 검증한다.

### 1.2 사용 데이터

| 항목 | 내용 |
|------|------|
| 원본 데이터 | AIHub 1339 콜센터 상담 스크립트 (감염병 QA 약 5,691개) |
| 샘플링 방법 | 전체 QA에서 약 10% 무작위 샘플링 |
| 최종 데이터 | **571개** (유사도 ≥ 0.6 → 527개 / 미달 → 44개) |
| 파일 | `데이터/similarity_sample_570.json` |

### 1.3 평가 방법
1. 각 QA의 질문(question)을 `text-embedding-3-small`로 임베딩
2. Knowledge Base 전체 청크와 코사인 유사도 계산
3. Top-1 유사도 값 기록
4. 임계값 0.6 기준으로 Pass / Fail 판정

### 1.4 평가 코드
- `코드/run_similarity_all.py`

### 1.5 결과

| 지표 | 값 |
|------|------|
| 샘플 크기 | 571개 |
| 유사도 ≥ 0.6 | 527개 (**92.3%**) |
| 유사도 < 0.6 | 44개 (7.7%) |

> **해석:** 실제 상담 발화의 92.3%가 RAG 검색 임계값 이상으로 관련 문서를 검색할 수 있음.

## 2. 라우팅(의도분류) 평가

### 2.1 목적
고객 발화에 대해 LLMSession이 올바른 카테고리로 분류하는지 정확도를 측정한다.

### 2.2 사용 데이터

| 항목 | 내용 |
|------|------|
| 원본 데이터 | AIHub 1339 콜센터 상담 스크립트 50개 대화 |
| 라벨링 방법 | 대화 카테고리 비율 유지하여 30개 선별 후 발화 단위 수동 라벨링 |
| 최종 데이터 | **30개 대화, 157개 발화** |
| 파일 | `데이터/llm_session_test_30_labeled.json` |

> **수동 라벨링 배경:** AIHub 데이터는 상담 세션 단위 카테고리만 제공하며,
> 발화 단위 의도 레이블이 없어 직접 라벨링 진행.

### 2.3 카테고리 분포 (30개 대화)

| 카테고리 | 대화 수 |
|----------|:---:|
| 감염병 | 14 |
| 예방접종 | 4 |
| 접수처리 (action_required) | 4 |
| 감염병 통계·현황 | 3 |
| 해외/검역 정보 문의 | 2 |
| 타기관 이관 (transfer) | 2 |
| 실시간·지역정보 | 1 |

### 2.4 평가 방법
1. 각 발화를 LLMSession.on_utterance()에 입력
2. 반환된 category / oos_type을 수동 라벨과 비교
3. Trigger ID F1, Trigger Classification F1, Category Accuracy 산출

### 2.5 평가 코드
- `코드/run_llm_session_eval_30.py`

### 2.6 결과

| 지표 | 값 |
|------|:---:|
| Category Accuracy | **84.3%** |
| Trigger ID F1 | **0.859** (P=0.798, R=0.930) |
| Trigger Classification F1 | **0.724** (P=0.672, R=0.783) |

**카테고리별 정확도**

| 카테고리 | 발화 수 | 정확도 |
|----------|:---:|:---:|
| 감염병 | 75 | 92.9% |
| 예방접종 | 29 | 82.8% |
| 감염병 통계·현황 | 22 | 73.7% |
| 타기관 이관 (transfer) | 9 | 100.0% |
| 접수처리 (action_required) | 13 | 61.5% |
| 실시간·지역정보 | 5 | 40.0% |
| 해외/검역 정보 문의 | 4 | 100.0% |

> **해석:** Trigger ID Recall 0.930 — 고객이 무언가를 묻거나 요청할 때 놓치지 않고 반응.
> Precision 0.798로 과잉 발화는 낮은 수준으로 제어됨.

## 3. LLM-as-Judge (응답 품질 평가)

### 3.1 목적
RAG가 생성한 답변이 실제 상담사 답변(Gold) 대비 어느 수준인지 평가한다.

### 3.2 사용 데이터

| 항목 | 내용 |
|------|------|
| 원본 데이터 | AIHub 1339 콜센터 상담 스크립트 (감염병 QA) |
| 최종 데이터 | **407개** QA 쌍 (질문 + 상담사 답변 + RAG 생성 답변) |
| 파일 | `데이터/test_qa_dataset_v6_filtered.json` |

### 3.3 평가 방법
1. 각 질문에 대해 RAG 답변 생성 (Hybrid RAG + card_generator)
2. GPT-4o-mini를 Judge 모델로 사용
3. Gold 답변(상담사)과 RAG 답변 각각에 대해 4개 항목 1~5점 채점
4. 점수 비교

**채점 기준**

| 항목 | 설명 |
|------|------|
| accuracy (정확성) | 질문에 대한 내용이 사실적으로 맞는가 |
| completeness (완전성) | 핵심 정보를 충분히 담았는가 |
| usefulness (유용성) | 실제 콜센터 상담에서 바로 활용할 수 있는 수준인가 |
| overall (종합) | 전체적인 답변 품질 |

### 3.4 평가 코드
- `코드/eval_response.py`

### 3.5 결과

**RAG 답변 생성 현황**

| 상태              |   수  |   비율  |
| --------------- | :--: | :---: |
| 성공 (success)    | 363개 | 89.2% |
| 미검색 (no_result) |  44개 | 10.8% |


**항목별 점수 비교 (1~5점)**

| 항목 | Gold (상담사) | RAG | 차이 |
|------|:---:|:---:|:---:|
| accuracy (정확성) | 4.172 | 4.523 | **+0.351** |
| completeness (완전성) | 3.201 | 3.705 | **+0.504** |
| usefulness (유용성) | 3.764 | 4.107 | **+0.343** |
| **overall (종합)** | **3.703** | **4.039** | **+0.336** |

**Overall 비교 분포 (전체 407개)**

| 구분         |   수  |   비율  |
| ---------- | :--: | :---: |
| RAG > Gold | 158개 | 38.8% |
| RAG = Gold | 148개 | 36.4% |
| RAG < Gold | 101개 | 24.8% |


> **해석:** RAG 답변이 전 항목에서 상담사 답변 대비 높은 점수를 기록.
> 특히 완전성(+0.504)에서 가장 큰 차이 — 상담사 답변이 구어체·요약 중심인 반면,
> RAG는 지침 문서 기반으로 체계적인 정보를 제공하기 때문.

## 4. STT 인식 품질 평가

### 4.1 목적
Deepgram 기반 STT 모듈의 음성 인식 정확도와 응답 지연 시간을 측정하여
실시간 콜센터 상담 환경에서의 사용 가능성을 검증한다.

### 4.2 모델 선택 배경

실시간 스트리밍 STT 구현을 위해 Whisper와 Deepgram Nova-2를 비교 평가하였으며,
실제 시연 환경 기준으로 CER을 측정하였다.

| 항목 | Whisper | Deepgram Nova-2 |
|------|:---:|:---:|
| CER | **9.9%** | **13.7%** |
| 실시간 스트리밍 지원 | 미지원 (배치 처리) | 지원 |

> **선택 근거:** Deepgram은 CER이 소폭 높으나, WebSocket 기반 실시간 스트리밍을 지원하여
> 콜센터 상담사가 고객 발화 중 즉시 텍스트를 확인할 수 있는 실시간 보조 환경 구현에 적합하다.

> **화자분리(Speaker Diarization) 불필요:** 본 시스템은 콜센터 듀얼 채널 음성을 사용하므로
> 상담사/고객 채널이 물리적으로 분리되어 있어 별도의 화자분리 처리가 필요 없다.

### 4.3 평가 방법
- CER: 모델 선택 단계에서 Whisper와 Deepgram을 시연 환경 기준으로 비교 측정
- Latency: 실제 시연 중 음성 입력 시작부터 텍스트 반환까지 소요 시간 관측 (ms)
- 별도의 공식 벤치마크 스크립트 없이 개발·시연 과정에서 수집한 수치

### 4.4 결과

**CER (Character Error Rate)**

| 모델 | CER |
|------|:---:|
| Deepgram Nova-2 (최종 채택) | **13.7%** (질병명 매핑 후처리 적용으로 추가 개선) |
| Whisper (비교군) | 9.9% |

**Latency**

| 지표 | 값 |
|------|:---:|
| 평균 응답 시간 | **138ms** |

> **해석:** 평균 138ms로 실시간 상담 보조 시스템의 응답성 요구사항을 충족함.
> 콜센터 상담사가 고객 발화 직후 즉시 텍스트 변환 결과를 확인할 수 있는 수준.
