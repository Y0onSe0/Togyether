# 2. 화면 전환 흐름

```
[SCR-001 로그인]
    │ 성공
    ▼
[SCR-003 대기 중 상태] ◀─────────────────┐
    │ 상담 시작 버튼                       │ ACW 저장 완료
    ▼ (상태 전환, 화면 이동 X)             │
[SCR-003 상담 중 상태]                    │
    │ 통화 종료                           │
    ▼ (화면 이동)                         │
[SCR-004 ACW 카드 작성] ─────────────────┘

[SCR-003/004/005] → GNB 상담사명 클릭
    └─ 팝오버 표시 (화면 이동 X)
         └─ 로그아웃 → [SCR-001]

[SCR-003] → LNB → [SCR-005 대시보드]
                        └─ 서브 탭 → 내 통계 / 전체 통계

[SCR-001] → 계정 생성 링크 → [SCR-002] → 완료 후 [SCR-001]
```

### 3.4.3 SCR-004 액션 흐름

```
통화 종료
  → SCR-004 자동 전환 + ACW 타이머 시작 (ACW-001)

  [Step A: 자동]
  → conversation_history → transcript 포맷 변환
       "[HH:MM:SS] 상담사/고객: ..." → acw_cards.transcript 저장
  → ai_guidance 캐시 조회
       ai_guidance.answer  → Section 4 AI 생성 답변 표시
       ai_guidance.sources[] → Section 4 참고 문서 ①②③ 표시

  [Step B: 자동, Step A 완료 후]
  → ACW LLM 1회 호출 (gpt-4o-mini) (ACW-002)
       입력: transcript + ai_guidance 캐시
       생성 필드 pre-fill:
         ├─ Section 2: 제목, 고객 유형, 카테고리 계열, 질병명, 이관 여부
         ├─ Section 3: AI 상담 요약 (단일 서술형 단락)
         └─ Section 5: Q/A 쌍

  [Step C: 자동]
  → GET /api/categories → 드롭다운 데이터 로드

  [상담사 검토 및 수정]
  → Section 2: 통화 기본 정보 자유 편집
  → Section 4: 읽기 전용 (수정 불가)
  → Section 5: Q/A 수정 가능
  → Section 6: 해결 여부 / AI 활용 여부 선택 (필수)
  → Section 7: 메모 입력 (선택)

  [임시저장 버튼]
  → PUT /api/acw/{call_id} (필수값 미검증)

  [완료 버튼]
  → 필수값 검증
       ├─ 미입력 → 해당 필드 오류 표시, 저장 차단
       └─ 통과
           → Q 텍스트 → q_embedding 생성 (text-embedding-3-small)
           → acw_cards 전체 저장
           → calls.status = 'ended' 업데이트
           → SCR-003 (대기 중) 복귀
```
